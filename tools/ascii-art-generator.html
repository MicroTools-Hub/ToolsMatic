Text Conversion Questions

      
How do I keep lowercase letters in my ASCII art?

      
In the "üì§ Case" dropdown, select "Preserve input" to maintain your original text casing in the output. By default, the tool converts everything to uppercase for cleaner grid alignment, but you have full control.


      
What's the best text length for ASCII art generation?

      
Short titles (5-20 characters) work best for terminal readability and visual impact. Long text (50+ characters) will generate wide output. For best results in terminals and chat apps, keep lines under 80 characters. Create multiple lines of separate ASCII conversions if needed.


      
How do I use the letter spacing control?

      
The spacing slider (0-5) controls the gap between characters. Set it to 0 for compact, tight text. Increase it to 3-5 for more readable, airy output. Experiment to find the balance that works for your design.


      
Can I combine multiple ASCII arts?

      
Yes! Generate multiple separate conversions, then manually combine them by pasting the ASCII output together. Many users create multi-line banners by converting different parts separately and stacking them.

What do the Density and Contrast sliders do?

      
Density (1-10) controls how many characters are used‚Äîhigher values create more detailed ASCII art with more characters per pixel. Contrast (0-100%) adjusts brightness levels to improve the ASCII output quality and make details more visible.


      
What's the difference between character sets in image conversion?

      
Different character sets produce different visual effects: Full (70 chars) = most detailed, Standard (50 chars) = balanced, Simple (15 chars) = bold & abstract, Blocks (8 chars) = minimal geometric shapes.

Troubleshooting

      
Why are some characters showing as "?"?

      
The "?" character indicates an unsupported character in your input. The tool supports A-Z, 0-9, and common punctuation. Check for special unicode characters or symbols that aren't in the standard ASCII character set and replace them.


      
The ASCII art doesn't look right when I paste it elsewhere. Why?

      
Make sure you're using a monospace font in your target application (noted as "Courier", "Monaco", "Consolas", or similar). ASCII art requires monospace fonts to render properly. If using Markdown, wrap the output in triple backticks.


      
Why is the output different sizes on mobile vs desktop?

      
The responsive design adjusts font sizes for readability on different screens. The actual ASCII character count stays the same, but the visual rendering may appear different due to viewport sizing. The monospace output is always preserved correctly.


      
The image conversion isn't working. What should I try?

      
Make sure your image is under 5MB, in a supported format (JPG, PNG, GIF, WebP), and has reasonable contrast. Very light or very dark images may not convert well. Adjust the contrast slider higher (70-100%) for better results.
